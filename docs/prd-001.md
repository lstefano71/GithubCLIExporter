# Copilot CLI Session Exporter — PRD-001

This document describes the export functionality for Copilot CLI sessions. The export feature allows users to save their session data, including commands, outputs, and interactions, in a human readable format for future reference or sharing. 

## Export Format
The session should be exported as:
1) Markdown — single self-contained `.md` file
2) HTML — single self-contained `.html` file with inline CSS, dark/light theme toggle, syntax highlighting

## Session Format (Discovered)

### Session Storage Location
Sessions are stored in `~/.copilot/session-state/<uuid>/` where `<uuid>` is an opaque GUID. The directory name alone is not human-readable, so the tool must support session discovery and convenient selection.

### Session Directory Structure
| File/Dir | Purpose |
|---|---|
| `workspace.yaml` | Session metadata: `id`, `cwd`, `git_root`, `repository`, `summary`, `summary_count`, `created_at`, `updated_at` |
| `events.jsonl` | Main event stream — one JSON object per line (~3MB for a medium session, 1000+ events) |
| `session.db` | SQLite database with `todos` (id, title, description, status, created_at, updated_at) and `todo_deps` (todo_id, depends_on) tables |
| `plan.md` | Session plan created by the agent (if any) |
| `vscode.metadata.json` | VS Code metadata (typically empty `{}`) |
| `checkpoints/` | Checkpoint markdown files + `index.md` — summarize work completed at various points |
| `research/` | Research artifacts (may be empty) |
| `files/` | Persistent session files |
| `rewind-snapshots/` | `index.json` with snapshot history — maps to git commits and file backups |

### Event Schema
Every event has: `type` (string), `data` (object), `id` (UUID), `timestamp` (ISO 8601), `parentId` (UUID or null). Events form a tree via `parentId`.

**Session lifecycle:**

### Tool Names Observed
`create`, `edit`, `exit_plan_mode`, `github-mcp-server-get_commit`, `glob`, `grep`, `list_powershell`, `powershell`, `read_powershell`, `report_intent`, `sql`, `task`, `task_complete`, `view`, `web_fetch`

### Assistant Message Patterns
- Messages may have text `content`, `toolRequests`, or both
- In the sample session: 80 messages with both, 80 with tools only, 6 with text only
- `content` may contain thinking/reasoning sections that should be made collapsible in the export

## Features

### Session Discovery & Selection
Since session directories have opaque UUID names, the tool provides:
1. **List command**: `copilot-export list` — scans all sessions, displays a numbered table sorted newest-first with columns: index, date, repository, summary
2. **Filters**: `--repo <pattern>`, `--since <date>`, `--search <text>` (searches summary)
3. **Partial UUID prefix** (git-style): e.g. `copilot-export export 658a` matches `658af01e-...` if unique; errors if ambiguous
4. **Index number**: e.g. `copilot-export export 3` picks the 3rd session from the latest scan
5. **Interactive TUI picker**: if no session specified, launches a scrollable table with arrow-key navigation and `/` search

### Export Format Design

#### Conversation View
The export renders a chronological conversation view:
- **User messages** displayed prominently
- **Assistant responses** with text content rendered directly
- **Thinking sections** wrapped in collapsible `<details>` tags
- **Tool calls** wrapped in collapsible `<details>` with tool name, arguments, and output
- **Sub-agent interactions** wrapped in collapsible `<details>`
- **Session errors** shown as inline warnings

#### Metadata & Context
- **Header**: repository name, working directory, timestamps, Copilot version, mode from `workspace.yaml` + session events
- **Plan section**: contents of `plan.md` if present
- **Todos table**: from `session.db` with status indicators (✅ done, ⏳ in progress, etc.)
- **Checkpoints appendix**: summaries from `checkpoints/`
- **Session statistics footer**: premium requests, API duration, code change summary from shutdown event

#### HTML-Specific
- Self-contained single file (CSS inlined, no external dependencies)
- Dark/light theme toggle (CSS-only)
- Syntax highlighting for code blocks
- Table of contents with anchor links
- Responsive layout

## Implementation

### Phase 1: Python CLI Prototype
Technology: Python 3.10+ with `pyyaml`, `rich`, `markdown`, `pygments`.

Self-contained executable not a goal for Phase 1; focus on correctness and iteration speed.

### Phase 2: C# AOT Binary (Future)
Technology: C# 14 / .NET 10+ / AOT compilation for a self-contained single-file executable.

Port the Python logic using:
- `System.Text.Json` for JSONL parsing
- `Microsoft.Data.Sqlite` for session.db
- `YamlDotNet` for workspace.yaml
- AOT-compatible single-file publish
